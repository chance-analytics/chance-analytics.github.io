    <h3>

      <a href="https://github.com/chance-analytics/california-wildfire-containment-housing-exposure-modeling" target="_blank" rel="noopener">

        California Wildfire Containment & Housing Exposure Modeling

      </a>

    </h3>

    <p>

      I built an end-to-end analysis in a Jupyter Notebook to explore California wildfire records and estimate two outcomes:

      wildfire containment duration and potential housing value exposure by linking wildfire patterns with Zillow home value data.

      The work included data cleaning and EDA, then a baseline Linear Regression model and a stronger Random Forest Regressor,

      evaluated with regression metrics (RMSE / R²).

    </p>

    <p><strong>Tools:</strong> Python, pandas, matplotlib/seaborn, scikit-learn (feature preprocessing, SelectKBest mutual information, PCA, LinearRegression, RandomForestRegressor).</p>

  <div class="card">

    <h3>

      <a href="https://github.com/chance-analytics/candlestick-chart-image-classification-short-term-price-movement" target="_blank" rel="noopener">

        Candlestick Chart Image Classification for Short-Term Price Movement (Computer Vision + Multi-Modal Fusion)

      </a>

    </h3>

    <p>

      This project asked whether a model could “read” candlestick chart images and predict short-term price movement.

      I built a reproducible pipeline using daily OHLCV data for SPY, AAPL, MSFT, and NVDA (2015–2025), created 3-class labels with

      a triple-barrier method (Up/Down/Hold using a 20-day horizon and ±5% barriers), and generated standardized candlestick PNG images

      (224×224) from rolling lookback windows. I trained a vision-only ResNet18 baseline and tested two multi-modal fusion variants that

      combined images with numeric indicators (MACD, RSI, Bollinger) using time-based splits. Results were modest (~0.45 test accuracy and

      macro-F1), with persistent confusion between Up and Down.

    </p>

    <p><strong>Tools:</strong> Python, PyTorch, ResNet18, time-based validation, technical indicators (MACD/RSI/Bollinger).</p>

  </div>

    <h3>

      <a href="https://github.com/chance-analytics/genai-stock-stance-dashboard-streamlit" target="_blank" rel="noopener">

        GenAI Stock Stance Dashboard (Fine-Tuning + Technical Indicators + Streamlit App)

      </a>

    </h3>

    <p>

      An end-to-end GenAI-assisted stock analysis system that scored a personal watchlist (~129 tickers) using engineered technical indicators

      plus a fine-tuned OpenAI “stance head” model. A Python pipeline pulled ~420 trading days of OHLCV data, computed indicators (RSI, MACD,

      Bollinger Bands, 52-week range, support/resistance, TD Sequential), generated a transparent rule-based local score, then called the

      fine-tuned model to produce an LLM stance, score, and summary. Finally, it blended local and LLM scores into a final decision surfaced

      in an interactive Streamlit dashboard with filtering, sorting, charts, and per-ticker drilldowns.

    </p>

    <p><strong>Tools:</strong> Python, yfinance, feature engineering, fine-tuned LLM, Streamlit dashboard.</p>

        Investment Strategy Research & Market Predictability (R + EDA)

  <div class="card">

    <h3>

      <a href="https://github.com/chance-analytics/night-vs-day-crash-severity-analysis" target="_blank" rel="noopener">

        Night vs. Day Crash Severity Analysis (ThinkStats2 / Python EDA + Hypothesis Testing)

      </a>

    </h3>

    <p>

      Using a Miami, FL crash dataset (2016–2023; ~186k records), this project tested whether crashes were more severe at night than during the day

      (defined using astronomical twilight). I performed data cleaning/outlier handling, EDA (histograms, PMF/CDF, correlations), and hypothesis

      testing (chi-square and one-sided mean-difference tests) in Python using ThinkStats2/thinkplot and statsmodels.

    </p>

    <p><strong>Tools:</strong> Python, pandas/NumPy, ThinkStats2, statsmodels, hypothesis testing.</p>

  <div class="card">

    <h3>

      <a href="https://github.com/chance-analytics/prediction-markets-spy-next-day-forecasting-polymarket" target="_blank" rel="noopener">

        Prediction Markets for Next-Day S&P 500 Forecasting (Polymarket + Feature Engineering + Walk-Forward ML)

      </a>

    </h3>

    <p>

      This project tested whether Polymarket probabilities could be turned into useful signals for forecasting whether SPY went up or down the next trading day.

      I pulled market metadata (Gamma API), downloaded YES-token price histories (CLOB prices-history), converted them into daily probability features, and aligned

      them to U.S. trading days alongside SPY and VIX. Because “SPX up/down tomorrow” markets were sparse, I pivoted to a broader set of finance/econ markets and

      aggregated them into topic signals (e.g., crypto, rates) to reduce missingness. Using walk-forward validation and elastic-net logistic regression, I found

      limited accuracy lift but a meaningful AUC improvement after adding VIX, suggesting better ranking across regimes.

    </p>

    <p><strong>Tools:</strong> Python, API ingestion, feature engineering, walk-forward validation, elastic-net logistic regression.</p>

  </div>


  <div class="card">

    <h3>

      <a href="https://github.com/chance-analytics/stock-market-prediction-pipeline-modeling" target="_blank" rel="noopener">

        Stock Market Prediction Pipeline & Modeling (Python + SQL + Web Scraping + ETL + ML/NLP)

      </a>

    </h3>

    <p>

      An end-to-end stock market prediction pipeline focused on a daily up/down classifier for SPY. I automated ETL across three sources:

      FRED macro indicators, Yahoo Finance market data, and Wikipedia world events (web scraping/text). I standardized and aligned fields to a

      daily timeline, engineered features (EMA/RSI/MACD/Bollinger + macro variables + event-sentiment signals), stored the final dataset in SQLite,

      then trained and tuned Random Forest and XGBoost models with time-based splits (2020–2023 train, 2024 test). Performance was modest (~50–53%)

      out-of-sample, highlighting the difficulty of short-horizon market prediction.

    </p>

    <p><strong>Tools:</strong> Python, pandas, SQLite, web scraping, ETL, Random Forest, XGBoost, GridSearchCV.</p>
